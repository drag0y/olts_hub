## Установка v3.0
Клонировать репозиторий или скачать архив в любую удобную директорию.
Перейти в папку с приложением если ещё не там.
Создаём виртуальное окружение:
```commandline
python3 -m venv .venv
```
Где __.venv__ это директория, где и будет наше виртуальное окружение (она создастся при выполнении команды).
Для перехода в виртуальное окружение надо выполнить команду:
```commandline
source .venv/bin/activate
```
После успешной активации виртуального окружения, в командной строке рядом с именем пользователя будет надпись __(.venv)__. 

Установить необходимые модули из файла requirements.txt.
```commandline
pip install -r requirements.txt
```
Далее переименовать файл-шаблон конфигурации __env_template.py__ в __.env__.
```commandline
mv env_template.py .env
```

Для работы Web сервера Flask необходимо создать секретный ключ.\
Для этого запускаете скрипт __key_generate.py__
```commandline
python3 key_generate.py
```
И то, что выведется на экран копируете в файл __.env__ в поле **SECRET_KEY** 

При необходимости заполните/поменяйте остальные переменные.

При работе, программа использует snmpwalk. Поэтому, он дожен быть установлен в системе.\
В Ubuntu подобных системах это можно сделать так.
```commandline
sudo apt install snmp
```

## Первый запуск, создание новой базы
Перед первым запуском необходимо создать базу данных.\
В корневой директории приложения, необходимо запустить файл __create_db.py__
Данный скрипт создаст новую БД, если у вас уже была БД, то она удалится.
```commandline
python3 create_db.py
```
В случае успешного выполнения, на консоль будет выведено сообщение "Создана новая база данных:".\
Приложение работает с базой sqlite, для её работы, дополнительно устанавливать ничего не требуется. 
База данных представляет из себя файл с базой, в папке instance.

После чего можно запускать приложение.

```commandline
python3 app.py
```
В браузере вводите http://ip:5000 или другой порт, если поменяли его в __.env__

Дефолтный логин/пароль - __root/admin__.

Воспользуйтесь инструкцией для [Администратора](admin_guide.md) для первоначальной настройки, а инструкцию пользователя можно посмотреть в браузере в меню __Справка__

## Если была установлена предыдущая версия
Версия 3.0 не совместима с предыдущими версиями, т.к. имеет новую структуру БД.

Но можно перенести пользователей и ОЛТы. 
Для этого переименуйте старую базу __onulist.db__ в __backup.db__, создайте новую, 
и запустите скрипт 
```commandline
python3 migrate_users_olts.py
```
Скрипт переносит только пользователей и список ОЛТов, все остальные данные, типа SNMP community и NetBox, нужно будет прописать завново.
